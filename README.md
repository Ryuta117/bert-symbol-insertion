# bert-symbol-insertion

## 概要
BERTを用いて、ペン入力環境での丸括弧の自動挿入を行うモデルの実装。
括弧を除去した文章を入力として、括弧が挿入されるべき位置を予測する。

## 背景・目的
音声入力では、括弧などの記号を入力する操作が非効率である。

本研究では、この課題を解決する第一歩として、BERTを用いて文章中の各トークン間に
丸括弧が挿入される確率を出力するモデルを実装した。
具体的には、括弧を除去した文章をBERTに入力し、各トークン間のギャップごとに
開き括弧・閉じ括弧それぞれの挿入確率を予測する。
このモデルの出力確率を、ペン入力や音声入力のスコアと組み合わせることで、
最終的な括弧の自動挿入を実現することを目指している。

## 使用技術 (Tech Stack)
- **言語:** Python 3.11
- **フレームワーク:** PyTorch, PyTorch Lightning
- **モデル:** LINE DistilBERT Japanese
- **開発環境:** Paperspace (GPU環境)


## 機能一覧
- **データ収集・前処理:** HuggingFaceの日本語コーパスから括弧あり・なしの文を収集し、フィルタリング
- **GapProjector:** トークン間のギャップ表現を学習可能な重みで生成する独自モジュール
- **括弧挿入予測:** 開き括弧・閉じ括弧の挿入をバイナリ分類で予測


## 実験結果

| メトリクス | スコア |
|------------|--------|
| test_open_f1 | 0.764 |
| test_open_precision | 0.845 |
| test_open_recall | 0.697 |
| test_close_f1 | 0.678 |
| test_close_precision | 0.781 |
| test_close_recall | 0.600 |
| test_fp_open | 0.0022 |
| test_fp_close | 0.0029 |


## 工夫した点
- トークン間のギャップを予測単位とすることで、括弧の挿入位置を自然に扱えるよう設計
- 閉じ括弧の予測に開き括弧の予測確率を特徴量として追加し、括弧の対応関係を考慮
- GapProjectorにより、ギャップ左右のトークン表現を学習可能な重みで加重平均して特徴量を生成
- データ収集時に括弧の対応チェック・文字数フィルタリングなどの前処理パイプラインを自作
